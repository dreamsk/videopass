# VideoPass - 项目需求文档

## 1. 项目概述

VideoPass 是一个基于 Web 的视频提取工具，用户可以通过粘贴视频链接来提取视频信息、封面图，并下载视频文件。支持在线视频播放（倍速/全屏）、封面图保存、批量链接提取。

## 2. 功能需求

### 2.1 核心功能

| 编号 | 功能 | 描述 | 优先级 | 状态 |
|------|------|------|--------|------|
| F-01 | 链接解析 | 用户粘贴视频链接，系统自动识别平台 | P0 | [x] |
| F-02 | 视频信息提取 | 提取视频标题、封面图、时长、上传者 | P0 | [x] |
| F-03 | 封面图展示 | 在页面上展示视频封面缩略图 | P0 | [x] |
| F-04 | 格式选择 | 展示可用的视频格式/画质选项 | P0 | [x] |
| F-05 | 视频下载 | 用户选择格式后下载视频文件（自动合并视频+音频） | P0 | [x] |
| F-06 | 平台检测 | 根据 URL 自动识别来源平台并显示图标 | P1 | [x] |
| F-07 | 加载状态 | 提取和下载过程中显示加载动画 | P1 | [x] |
| F-08 | 错误处理 | 无效链接、不支持平台等情况的友好提示 | P1 | [x] |
| F-09 | 封面图保存 | 点击按钮保存视频封面图到本地 | P1 | [x] |
| F-10 | 在线视频播放 | 点击封面图播放视频，支持倍速和全屏 | P1 | [x] |
| F-11 | 批量链接提取 | textarea 输入多个链接，逐个提取（仅 Bilibili） | P1 | [x] |
| F-12 | 粘贴自动分行 | 粘贴多条粘连链接时自动识别并分行 | P2 | [x] |
| F-13 | 提取历史展示 | localStorage 持久化，点击恢复，删除/清空 | P1 | [x] |

### 2.2 支持平台

基于 yt-dlp，支持 1700+ 网站，重点支持以下平台：

| 平台 | URL 模式 | 优先级 | 状态 |
|------|----------|--------|------|
| YouTube | youtube.com, youtu.be | P0 | [!] |
| Bilibili | bilibili.com, b23.tv | P0 | [x] |
| TikTok | tiktok.com, vm.tiktok.com | P1 | [ ] |
| Twitter/X | twitter.com, x.com | P1 | [ ] |
| Instagram | instagram.com | P1 | [ ] |
| Vimeo | vimeo.com | P2 | [ ] |
| Twitch | twitch.tv | P2 | [ ] |
| 其他 | yt-dlp 支持的所有站点 | P2 | [ ] |

> `[!]` 表示受阻（当前环境网络无法访问 YouTube，非代码问题）

### 2.3 提取的视频信息字段

| 字段 | 描述 | 必需 |
|------|------|------|
| title | 视频标题 | 是 |
| thumbnail | 封面图 URL | 是 |
| duration | 视频时长（秒） | 是 |
| uploader | 上传者名称 | 否 |
| platform | 来源平台 | 是 |
| formats | 可用格式列表 | 是 |

### 2.4 格式信息字段

| 字段 | 描述 |
|------|------|
| format_id | 画质标签 (1080p, 720p, audio 等) |
| ext | 文件扩展名 (mp4, webm 等) |
| resolution | 分辨率标签 (1080p, 720p 等) |
| filesize | 文件大小（字节） |
| hasVideo | 是否包含视频流 |
| hasAudio | 是否包含音频流 |
| quality | 画质等级标签 |

## 4. 用户界面需求

### 4.1 页面布局

```
[Logo / 标题]
[副标题 / 说明]
[支持平台图标行]
[模式切换: 单个 / 批量]
[--- 单个模式 ---]
[URL 输入框 + 提取按钮]
[--- 批量模式 ---]
[Textarea 多行输入 + 批量提取按钮]
[--- 提取后显示 ---]
[视频预览卡片 (可多个)]
  - 封面图 (悬浮: 播放按钮 + 保存封面按钮)
  - 标题、上传者、时长
  - 格式选择器
  - 下载按钮
[--- 在线播放时 ---]
[视频播放器]
  - 播放/暂停
  - 进度条 (可拖动)
  - 倍速选择 (0.5x ~ 2x)
  - 音量控制
  - 全屏按钮
[错误提示区域]
```

### 4.2 交互流程

1. 用户粘贴视频链接到输入框（支持粘贴自动分行）
2. 系统实时检测平台并显示对应图标
3. 用户点击"提取"按钮（批量模式 Ctrl+Enter）
4. 显示加载骨架屏（批量模式显示进度 "正在提取 2/5..."）
5. 提取成功：显示视频卡片（封面、信息、格式选择）
6. 用户可点击封面图在线播放视频
7. 用户可点击封面图右下角按钮保存封面
8. 用户选择画质，点击"下载"
9. 浏览器开始下载视频文件（已自动合并视频+音频）

## 5. 技术约束

- 前后端使用 Next.js App Router 统一项目
- 视频提取依赖 yt-dlp（通过 child_process 直接调用）
- 视频合并依赖 ffmpeg（通过 chocolatey 安装）
- 需要 Python 3.9+ 环境（yt-dlp 依赖）
- 封面图通过 `/api/thumbnail` 代理加载（解决 CDN 403 问题）
- 在线播放通过 `/api/stream` 接口，支持 HTTP Range 请求
